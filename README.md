# Микросервис для отправки налоговых отчетов в ФНС

Этот проект представляет собой микросервис на Django для автоматизированной обработки и отправки налоговых отчетов. Он упакован в Docker для упрощения развертывания и обеспечения консистентности окружения.

## Требования

- [Docker](https://www.docker.com/products/docker-desktop/)

Убедитесь, что Docker Desktop установлен и запущен на вашей машине.

## Быстрый старт

Весь проект запускается одной командой из корневой директории.

### 1. Первый запуск и сборка

Эта команда соберет образ вашего Django-приложения, скачает образы PostgreSQL и Redis, создаст и запустит все контейнеры в фоновом режиме.

```bash
docker-compose up --build -d
```

- `--build`: Флаг для пересборки образ��, если вы внесли изменения в код или `requirements.txt`.
- `-d`: Запуск в фоновом (detached) режиме.

### 2. Проверка статуса контейнеров

Чтобы убедиться, что все сервисы (`web`, `worker`, `db`, `redis`) успешно запустились, выполните:

```bash
docker-compose ps
```

Вы должны увидеть, что у всех контейнеров статус `Up` или `running`.

### 3. Доступ к приложению

После успешного запуска, веб-приложение будет доступно в вашем браузере по адресу:

- **API:** [http://localhost:8000/api/reports/submit/](http://localhost:8000/api/reports/submit/) (и другие эндпоинты)
- **Админка Django:** [http://localhost:8000/admin/](http://localhost:8000/admin/) (потребуется создать суперпользователя)

### 4. Просмотр логов

Если что-то пошло не так или вы хотите отслеживать работу сервисов в реальном времени, вы можете посмотреть их логи.

```bash
# Посмотреть логи всех сервисов
docker-compose logs -f

# Посмотреть логи конкретного сервиса (например, web или worker)
docker-compose logs -f web
docker-compose logs -f worker
```

### 5. Остановка проекта

Чтобы остановить все запущенные контейнеры, выполните:

```bash
docker-compose down
```

Эта команда остановит и удалит контейнеры, но не затронет данные в базе данных, так как они хранятся в Docker Volume.

## Дополнительные команды

### Создание суперпользователя Django

Чтобы получить доступ к админ-панели, вам нужно создать суперпользователя.

```bash
docker-compose exec web python manage.py createsuperuser
```

### Выполнение миграций вручную

Миграции применяются автоматически при старте `web` контейнера. Если вам нужно применить их вручную:

```bash
docker-compose exec web python manage.py migrate
```
